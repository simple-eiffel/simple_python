Overview

simple_python is a production-ready Eiffel library for communicating with Python services via pluggable transport protocols. It provides HTTP/REST communication (with JSON serialization), IPC (Windows named pipes for ultra-low-latency), and extensible design for gRPC integration.

Unlike raw socket APIs that require extensive framing and serialization code, simple_python reduces typical Eiffel-Python bridge implementations from 50-100 lines to just 5-10 lines, while maintaining full control through Design by Contract.

Part of the Simple Eiffel ecosystem.

Quick Start

Installation

  1. Set the environment variable: 
    export SIMPLE_EIFFEL=D:\prod
  2. Add to your ECF file: 
    <library name="simple_python" location="$SIMPLE_EIFFEL/simple_python/simple_python.ecf"/>

HTTP Bridge Communication

local
    bridge: HTTP_PYTHON_BRIDGE
    request: PYTHON_VALIDATION_REQUEST
do
    create bridge.make_with_host_port ("localhost", 8080)

    create request.make ("validation_001")
    request.freeze

    if bridge.send_message (request) then
        if attached bridge.receive_message as response then
            print (response.to_json)
        end
    else
        print ("Error: " + bridge.last_error_message)
    end
end

Key Features

HTTP Bridge

JSON over HTTP/1.1 with configurable timeouts. Production-ready implementation with actual HTTP communication.

SCOOP Compatible

Thread-safe message handling with automatic freezing for concurrent access in SCOOP programs.

Design by Contract

Full preconditions, postconditions, and invariants on all classes and features.

Message Serialization

JSON and binary formats with 4-byte big-endian length prefix for reliable framing.

Pluggable Transports

HTTP, IPC (Windows named pipes), and gRPC via unified PYTHON_BRIDGE interface.

Void Safe

Complete null-safety guarantees with void_safety="all" throughout.

Architecture

simple_python follows the pluggable bridge pattern, allowing different transport protocols while maintaining a unified interface:

  • PYTHON_BRIDGE - Abstract interface defining bridge contract
  • HTTP_PYTHON_BRIDGE - HTTP/REST implementation (production-ready)
  • IPC_PYTHON_BRIDGE - Named pipes implementation (extensible)
  • GRPC_PYTHON_BRIDGE - gRPC implementation (Phase 2)
  • PYTHON_MESSAGE - Base for all message types
